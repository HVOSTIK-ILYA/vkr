# 2 Вёрстка и клиентская часть веб-сайта

Клиентская часть разработанного веб-сайта состоит из трёх страниц: главной страницы (index.php), страницы личного кабинета (account.php) и страницы оплаты (payment.html). Все страницы используют общий файл стилей style.css, что позволило избежать дублирования правил оформления и обеспечить единый внешний вид сайта. Поведение интерфейса — открытие модальных окон, маска ввода телефона, проверка формата адреса — реализовано средствами языка JavaScript, сценарии которого размещены непосредственно в разметке главной страницы.

В настоящей главе подробно рассматривается вёрстка каждой страницы и её логических элементов, применённые CSS-классы и их свойства, а также назначение использованных JavaScript-сценариев. Однотипные элементы (например, формы проверки задолженности или кнопки) описываются один раз, поскольку их устройство идентично.

## 2.1 Общие принципы построения разметки

Все страницы сайта построены по единой схеме. Документ начинается с объявления `<!DOCTYPE html>`, корневому элементу задан атрибут `lang="ru"`, указывающий язык содержимого. В секции `<head>` подключается кодировка UTF-8 и метатег viewport со значением `width=device-width, initial-scale=1.0`. Без этого метатега мобильные браузеры отображали бы страницу в уменьшенном «настольном» виде, и адаптивная вёрстка не работала бы. Там же подключается общий файл стилей:

```html
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Новосибирскэнергосбыт</title>
    <link rel="stylesheet" href="style.css">
</head>
```

Рисунок 2.1 — Общая структура секции head

При построении разметки использовались семантические теги HTML5: `<header>` для шапки сайта, `<main>` для основного содержимого, `<section>` для смысловых блоков внутри страницы и `<footer>` для подвала. Такой подход делает структуру документа понятной как для разработчика, так и для поисковых систем и программ экранного доступа: по тегу сразу видно назначение блока, в отличие от безликой вёрстки на одних `<div>`.

Базовое оформление страницы задаётся правилом для элемента `body`: обнуляется внешний отступ `margin`, который браузеры добавляют по умолчанию, устанавливается шрифтовая гарнитура Roboto с запасным вариантом sans-serif и минимальная ширина страницы 360 пикселей, ниже которой содержимое перестаёт сжиматься. Значение 360 пикселей выбрано как ширина экрана наиболее распространённых бюджетных смартфонов.

## 2.2 Вёрстка главной страницы

Главная страница является самой насыщенной по содержимому и состоит из следующих логических элементов: шапки, рекламного блока, ряда информационных кнопок, двух блоков проверки задолженности и набора модальных окон.

### 2.2.1 Шапка сайта

Шапка присутствует на всех страницах сайта и свёрстана одинаково, поэтому описывается один раз. Она представляет собой элемент `<header>` с классом `site-header`, внутри которого расположены два блока: область логотипа и контейнер с кнопкой входа в личный кабинет.

```html
<header class="site-header">
    <div class="logo-area">
        <img src="images/logo.png" alt="Логотип" class="logo-img">
        <span class="logo-text">Новосибирскэнергосбыт</span>
    </div>
    <div class="popout-container">
        <button id="accountBtn" class="account-Btn">Личный кабинет</button>
    </div>
</header>
```

Рисунок 2.2 — Разметка шапки сайта

Класс `site-header` превращает шапку во флекс-контейнер (`display: flex`). Свойство `justify-content: space-between` разводит логотип и кнопку по противоположным краям страницы, а `align-items: center` выравнивает их по вертикали. Внутренние отступы `padding: 10px 20px` не дают содержимому прилипать к границам окна.

Область логотипа `logo-area` также является флекс-контейнером: изображение и текстовая надпись выстраиваются в строку и выравниваются по центру, расстояние между ними задаётся свойством `gap: 10px`. Изображению логотипа (класс `logo-img`) принудительно заданы размеры 50 на 50 пикселей, чтобы исходный размер картинки не влиял на высоту шапки. Название компании оформлено классом `logo-text`: крупный кегль 37 пикселей, полужирное начертание, чёрный цвет.

Кнопка «Личный кабинет» с классом `account-Btn` стилизована в фирменной чёрно-белой гамме сайта: чёрный фон, белый текст, скруглённые на 10 пикселей углы, внутренние отступы 10 на 20 пикселей и кегль 1.5rem. Стандартная рамка кнопки убрана (`border: none`), а свойство `text-decoration: none` позволяет применять этот же класс к ссылкам — именно так в личном кабинете оформлена кнопка «Выйти», которая технически является тегом `<a>`. При наведении курсора срабатывает псевдокласс `:hover`: фон светлеет до серого `rgb(92, 92, 92)`, а курсор принимает вид указателя, подсказывая пользователю, что элемент кликабелен.

### 2.2.2 Рекламный блок и информационные кнопки

Сразу под шапкой расположен рекламный блок — секция с классом `block-1`, сообщающая о новой услуге «Автоплатёж за свет». Блок состоит из заголовка `<h1>` и иллюстрации. Фоном секции служит изображение, заданное через CSS-свойство `background-image`. Заголовок и картинка разнесены по горизонтали тем же приёмом, что и в шапке: флекс-контейнер с `justify-content: space-between`.

Особенность этого блока — широкое применение CSS-функции `clamp()`, которая позволяет задать «резиновое» значение с нижней и верхней границей. Например, расстояние между заголовком и картинкой описано как `gap: clamp(20px, 5vw, 120px)`: на узких экранах зазор не опустится ниже 20 пикселей, на широких не превысит 120, а между этими границами будет плавно меняться пропорционально ширине окна (5vw — пять процентов ширины области просмотра). Аналогично заданы внутренние отступы блока и размер шрифта. Такой приём избавляет от необходимости прописывать множество промежуточных контрольных точек в медиазапросах: блок сам подстраивается под любую ширину экрана.

Иллюстрация с классом `auto-plata` тоже масштабируется через `clamp()`: её ширина меняется от 200 до 570 пикселей, занимая в промежутке 40 % ширины родителя. Свойство `height: auto` сохраняет пропорции изображения, `border-radius` со значением из `clamp()` скругляет углы, а `flex-shrink: 0` запрещает флекс-контейнеру сжимать картинку сильнее заданного минимума.

Под рекламным блоком располагается ряд из четырёх информационных кнопок — секция `info-area`. В отличие от остальных блоков страницы, здесь применена не флексбокс-, а грид-раскладка:

```css
.info-area{
    display: grid;
    grid-template-columns: repeat(4, 1fr);
    gap: 0;
    margin-left: 20px;
    margin-right: 20px;
    border: black solid 1px;
    border-top: 0;
}
```

Рисунок 2.3 — Стили контейнера информационных кнопок

Запись `repeat(4, 1fr)` создаёт четыре колонки строго одинаковой ширины независимо от длины надписей на кнопках — для флексбокса равенство колонок пришлось бы обеспечивать дополнительными свойствами. Контейнер обведён общей рамкой, при этом верхняя граница убрана (`border-top: 0`), чтобы рамка не удваивалась на стыке с рекламным блоком, у которого уже есть нижняя граница.

Сами кнопки (класс `info-btn`) выглядят как продолжение страницы: белый фон, отсутствие рамки, полужирный шрифт 20 пикселей и крупные внутренние отступы `50px 20px`, делающие зону нажатия заметной. Визуальное разделение кнопок между собой достигается правыми границами, заданными по идентификаторам `#info-btn-1`, `#info-btn-2` и `#info-btn-3`, — у последней, четвёртой кнопки правая граница не нужна, так как её роль выполняет рамка контейнера. При наведении фон кнопки слегка затемняется полупрозрачным серым цветом `rgba(237, 237, 237, 0.797)`.

### 2.2.3 Блоки проверки задолженности

Центральная функциональная часть главной страницы — два блока, через которые пользователь может проверить и оплатить задолженность: «По адресу» и «По лицевому счёту». Блоки устроены зеркально и используют одни и те же классы, поэтому подробно разбирается первый из них, а затем указываются отличия второго.

Каждый блок представляет собой контейнер (`display-2` для проверки по адресу и `display-3` для проверки по лицевому счёту) с рамкой и внешними отступами 20 пикселей. Контейнер делится на две части: презентационную секцию с заголовком и тематической иллюстрацией (классы `block-2` и `block-3`) и форму с полями ввода (классы `payment-area-1` и `payment-area-2`).

Презентационная секция — флекс-колонка с центрированием по обеим осям: свойства `flex-direction: column`, `align-items: center` и `justify-content: center` ставят заголовок над картинкой ровно по центру секции, а `gap: 25px` задаёт расстояние между ними. Заголовок оформлен классом `h1` с очень крупным кеглем 5rem, иллюстрация (класс `img-block`) имеет фиксированный размер 300 на 300 пикселей. Свойство `flex: 1` указывает, что секция занимает одну долю ширины контейнера; форме задано такое же значение, поэтому обе части делят блок пополам.

Форма свёрстана по следующей схеме:

```html
<form class="payment-area-1" id="form-adress" method="POST">
    <div class="columns">
        <section class="column-1">
            <img class="img-column-1" src="images/punkt-1.png">
            <label for="addressInput" class="label-for-form">Населенный пункт,
                улица (микрорайон, квартал, ДНП, иное), номер дома</label>
            <input class="forma" type="text" id="addressInput" name="address"
                placeholder="Введите адрес" required>
        </section>
        <section class="column-2"> ... </section>
    </div>
    <div class="btn">
        <button class="btn-1" type="submit">Оплатить</button>
    </div>
</form>
```

Рисунок 2.4 — Структура формы проверки задолженности

Внутри формы блок `columns` выстраивает две колонки в строку (`flex-direction: row`), каждая колонка (`column-1`, `column-2`) — это вертикальный флекс-контейнер, в котором сверху вниз расположены пиктограмма, текстовая подпись и поле ввода. Свойство `justify-content: space-between` равномерно распределяет эти три элемента по высоте колонки. Подписи к полям оформлены классом `label-for-form` (выравнивание по центру, кегль 1.5rem) и связаны с полями атрибутом `for`, благодаря чему щелчок по подписи переводит фокус в соответствующее поле.

Все поля ввода используют общий класс `forma`. Им задана пунктирная серая рамка (`border-style: dotted`), скругление 10 пикселей, центрирование вводимого текста и серый цвет символов. Тем же серым цветом окрашен текст подсказки внутри пустого поля — за это отвечает псевдоэлемент `input::placeholder`. При получении фокуса срабатывает псевдокласс `input:focus`: стандартная браузерная обводка убирается (`outline: none`), а пунктирная рамка становится сплошной чёрной, наглядно показывая активное поле.

Помимо стилизации, к полям применена встроенная HTML-валидация. Поле номера квартиры ограничено атрибутами `maxlength="3"` и `pattern="\d{1,3}"`, то есть принимает от одной до трёх цифр; атрибут `inputmode="numeric"` дополнительно подсказывает мобильным браузерам открыть цифровую клавиатуру. Поле номера лицевого счёта во втором блоке аналогично ограничено шестью цифрами. Атрибут `required` у всех полей запрещает отправку формы с пустыми значениями — браузер сам выводит подсказку, не допуская обращения к серверу с неполными данными.

Кнопка отправки «Оплатить» (класс `btn-1`) повторяет фирменный стиль кнопки личного кабинета: чёрный фон, белый текст, скругление и увеличение внутренних отступов до `15px 25px`. Кнопка центрируется внутри обёртки `btn` — флекс-контейнера, выравнивающего содержимое по центру обеих осей.

Второй блок, «По лицевому счёту», отличается от первого тремя деталями. Во-первых, контейнеру `display-3` задано свойство `flex-direction: row-reverse`, которое меняет порядок частей на обратный: форма оказывается слева, а иллюстрация справа. За счёт этого два одинаковых по устройству блока выглядят на странице зеркально, что делает композицию разнообразнее. Во-вторых, вместо текстового поля района используется выпадающий список `<select>` с десятью районами города Новосибирска; стандартная стрелка списка скрыта свойством `appearance: none`, чтобы список выглядел единообразно во всех браузерах и не выбивался из общего стиля полей. В-третьих, как уже отмечалось, поле лицевого счёта проверяется шаблоном `pattern="\d{6}"` ровно на шесть цифр.

### 2.2.4 Модальные окна

Значительная часть интерфейса главной страницы реализована через модальные окна — всплывающие блоки, перекрывающие страницу полупрозрачным затемнением. На странице используется восемь таких окон: четыре информационных (по одному на каждую информационную кнопку), окно ввода телефона для входа в личный кабинет, окно ввода кода подтверждения из приложения, окно с предложением установить приложение и окно с сообщением об отсутствии задолженности.

Все окна построены по одной схеме: внешний контейнер с классом `info-overlay` и внутренний блок с классом `info-box`.

```css
.info-overlay {
    position: fixed;
    inset: 0;
    z-index: 9999;
    background: rgba(0,0,0,0.6);
    display: none;
    align-items: center;
    justify-content: center;
}
.info-overlay.open { display: flex; }
```

Рисунок 2.5 — Стили затемняющей подложки модального окна

Контейнер `info-overlay` закреплён поверх страницы свойством `position: fixed`; сокращённая запись `inset: 0` обнуляет отступы со всех четырёх сторон, растягивая подложку на весь экран. Большое значение `z-index: 9999` гарантирует, что окно перекроет любой другой элемент страницы. Затемнение создаётся полупрозрачным чёрным фоном `rgba(0,0,0,0.6)`. По умолчанию окно скрыто (`display: none`); при добавлении класса `open` контейнер становится флекс-контейнером и центрирует содержимое по горизонтали и вертикали. Переключением класса `open` управляет JavaScript, который рассматривается в подразделе 2.6.

Внутренний блок `info-box` оформлен как тёмная карточка: чёрный фон, белый текст, внутренние отступы `30px 40px`, скругление 16 пикселей и мягкая тень `box-shadow: 0 10px 40px rgba(0,0,0,0.5)`, создающая эффект приподнятости над страницей. Максимальная ширина карточки ограничена 460 пикселями, чтобы на широких экранах окно не растягивалось во всю ширину. Заголовок окна набран кеглем 28 пикселей, перечни услуг оформлены нумерованными списками `<ol>` с увеличенным межстрочным интервалом 1.9 для удобства чтения.

В окнах с формами (ввод телефона и кода подтверждения) дополнительно используются класс `modal-submit` для кнопки отправки — белая кнопка на тёмном фоне карточки, инверсная по отношению к основным кнопкам сайта, — и правило `.info-box input`, убирающее рамку у полей ввода внутри окна. Кнопке «Скачать», которая является ссылкой, тот же класс `modal-submit` придаёт вид кнопки за счёт свойств `display: inline-block` и `text-decoration: none`.

Часть модальных окон выводится не скриптом, а сервером: PHP-условия `if ($showCode)`, `if ($showMaxCode)`, `if ($showDownload)` и `if ($message)` добавляют разметку окна в страницу только тогда, когда это требуется по логике работы (например, после отправки формы с найденной задолженностью). Такие окна сразу содержат класс `open` и видны с момента загрузки страницы, а атрибут `autofocus` у поля кода немедленно ставит в него курсор, избавляя пользователя от лишнего щелчка.

## 2.3 Вёрстка страницы личного кабинета

Страница личного кабинета доступна только после успешного входа и отображает список задолженностей пользователя. Её шапка повторяет шапку главной страницы с единственным отличием: вместо кнопки «Личный кабинет» выводится ссылка «Выйти», ведущая на сценарий завершения сессии. Ссылка оформлена тем же классом `account-Btn`, поэтому внешне неотличима от кнопки.

Основное содержимое размещено в элементе `<main>` с классом `account-main`. Это узкая центрированная колонка: максимальная ширина ограничена 800 пикселями, а внешние отступы `margin: 40px auto` прижимают колонку к центру окна — приём с автоматическими горизонтальными отступами является стандартным способом центрирования блока известной ширины. Боковые внутренние отступы 20 пикселей не дают содержимому касаться краёв экрана на мобильных устройствах. Заголовок «Ваши задолженности» оформлен классом `account-title` с кеглем 28 пикселей.

Список задолженностей формируется сервером: PHP-цикл перебирает строки результата запроса и для каждой выводит карточку.

```html
<div class="debt-cards">
    <?php while ($row = mysqli_fetch_assoc($result)): ?>
        <div class="debt-card">
            <p class="debt-info"><?= htmlspecialchars($row['details']) ?></p>
            <p class="debt-amount"><?= number_format($row['amount'], 2, '.', ' ') ?> ₽</p>
        </div>
    <?php endwhile; ?>
</div>
```

Рисунок 2.6 — Формирование карточек задолженностей

Контейнер `debt-cards` — вертикальный флекс-контейнер с зазором 16 пикселей между карточками. Каждая карточка `debt-card` выдержана в общем тёмном стиле сайта: чёрный фон, белый текст, скругление 14 пикселей и тень. Внутри карточки описание задолженности и её сумма разведены по краям свойством `justify-content: space-between` и выровнены по вертикали. Сумма (класс `debt-amount`) выделена увеличенным кеглем 22 пикселя, полужирным начертанием и красноватым цветом `#ff6b6b`, который хорошо читается на тёмном фоне и сразу привлекает внимание к размеру долга. У абзацев внутри карточки обнулены внешние отступы, чтобы браузерные отступы абзацев по умолчанию не раздували высоту карточки.

Если задолженностей у пользователя нет, вместо карточек выводится абзац с классом `no-debt` — зелёная надпись «Задолженностей нет», центрированная по горизонтали и отодвинутая от заголовка отступом 40 пикселей. Зелёный цвет здесь выбран намеренно как противоположность красному цвету сумм: отсутствие долга — положительное состояние.

## 2.4 Вёрстка страницы оплаты

Страница оплаты завершает сценарий погашения задолженности. В рамках выпускной квалификационной работы подключение реального платёжного шлюза не выполнялось, поэтому страница представляет собой заглушку: шапку сайта и центрированное сообщение. Тем не менее страница свёрстана по общим правилам — подключает тот же файл style.css и использует ту же разметку шапки, поэтому при дальнейшем развитии проекта в неё достаточно встроить платёжную форму, не меняя каркас.

Содержимое страницы оформлено классом `payment-page`, объявленным во встроенном блоке `<style>` прямо в документе, поскольку этот класс больше нигде не используется и выносить его в общий файл стилей нецелесообразно. Класс задаёт флекс-контейнер с центрированием по обеим осям, минимальную высоту 60 % высоты окна (`min-height: 60vh`) и крупный полужирный шрифт 36 пикселей — сообщение оказывается посередине видимой области экрана.

## 2.5 Адаптивность вёрстки

Сайт корректно отображается как на настольных мониторах, так и на экранах мобильных устройств. Адаптивность достигается двумя механизмами: уже описанной функцией `clamp()`, плавно меняющей размеры в заданных пределах, и медиазапросом `@media (max-width: 700px)`, который перестраивает раскладку при ширине окна 700 пикселей и менее. Порог 700 пикселей выбран опытным путём: именно на этой ширине двухколоночные блоки перестают помещаться рядом.

Внутри медиазапроса выполняется несколько групп изменений. Первая группа скрывает второстепенные элементы свойством `display: none`: текстовое название компании в шапке (остаётся только логотип), рекламную иллюстрацию и декоративные пиктограммы в блоках проверки задолженности. На маленьком экране эти элементы занимали бы место, не неся функциональной нагрузки, — пользователю мобильной версии важны прежде всего формы.

Вторая группа меняет направление флекс-контейнеров с горизонтального на вертикальное:

```css
.display-2, .display-3 {
    flex-direction: column;
}
.columns {
    flex-direction: column;
    gap: 20px;
}
```

Рисунок 2.7 — Перестроение блоков в одну колонку на узких экранах

Благодаря этим правилам блоки проверки задолженности, на широком экране состоящие из двух частей в ряд, на телефоне выстраиваются в одну колонку: заголовок, затем поля формы одно под другим, затем кнопка. Сетка информационных кнопок аналогично переводится из четырёх колонок в одну заменой значения `grid-template-columns` на `1fr`; правые границы кнопок при этом заменяются верхними, чтобы разделители соответствовали новому, вертикальному расположению.

Третья группа уменьшает размеры: заголовки блоков сокращаются с 5rem до 2.5rem, подписи и поля форм — с 1.5rem до 1.2rem, внутренние отступы информационных кнопок уменьшаются с `50px 20px` до `20px 5px`. Рекламный блок переводится в вертикальную компоновку с центрированным текстом, а размер его шрифта задаётся в относительных единицах `4vw`, привязанных к ширине экрана.

## 2.6 JavaScript-сценарии

Поведение интерфейса реализовано на чистом JavaScript без подключения сторонних библиотек. Все сценарии размещены в конце страницы index.php, что гарантирует их выполнение после построения DOM-дерева: к моменту запуска скрипта все элементы, к которым он обращается, уже существуют. На сайте используются четыре сценария: управление модальными окнами, маска ввода номера телефона, автоматическое сокращение слов в поле адреса и проверка формата адреса перед отправкой формы.

### 2.6.1 Управление модальными окнами

Открытие окон реализовано предельно лаконично: каждой кнопке назначается обработчик щелчка, который добавляет соответствующему окну класс `open`.

```javascript
document.getElementById('accountBtn').onclick =
    () => document.getElementById('modal-phone').classList.add('open');
document.getElementById('info-btn-1').onclick =
    () => document.getElementById('modal-1').classList.add('open');
```

Рисунок 2.8 — Обработчики открытия модальных окон

Как было показано в подразделе 2.2.4, класс `open` меняет значение свойства `display` контейнера окна с `none` на `flex`, и окно появляется на экране. Таким образом, вся видимая логика «показать/скрыть» сосредоточена в CSS, а скрипт лишь переключает класс — это разделение ответственности упрощает сопровождение кода.

Закрытие окон выполняется щелчком по затемнённой подложке. Один общий обработчик назначается всем окнам сразу через перебор коллекции `document.querySelectorAll('.info-overlay')`. Внутри обработчика проверяется условие `e.target === overlay`: свойство `target` объекта события содержит элемент, по которому фактически щёлкнул пользователь, и сравнение с самим контейнером позволяет отличить щелчок по тёмному фону от щелчка по содержимому окна. Если пользователь попал по фону, у окна снимается класс `open` и оно скрывается; щелчки внутри карточки окно не закрывают, что защищает от случайного закрытия при заполнении формы.

### 2.6.2 Маска ввода номера телефона

Поле телефона в окне входа в личный кабинет снабжено маской, которая на лету приводит любой ввод к единому формату «+7 (000) 000-00-00». Это важно для работы системы: телефоны хранятся в базе данных именно в таком виде, и без маски пользователь, набравший номер слитно или через восьмёрку, не был бы найден при проверке.

```javascript
phoneInput.addEventListener('input', function () {
    let d = this.value.replace(/\D/g, '');
    if (d[0] === '7' || d[0] === '8') d = d.slice(1);
    d = d.slice(0, 10);
    let r = '+7';
    if (d.length >= 1) r += ' (' + d.substring(0, 3);
    if (d.length >= 4) r += ') ' + d.substring(3, 6);
    if (d.length >= 7) r += '-' + d.substring(6, 8);
    if (d.length >= 9) r += '-' + d.substring(8, 10);
    this.value = r;
});
```

Рисунок 2.9 — Сценарий маски телефонного номера

Обработчик события `input` срабатывает при каждом изменении поля. Сначала регулярное выражение `/\D/g` удаляет из введённой строки всё, кроме цифр, — пользователь может вставить номер с пробелами, скобками или дефисами, и они будут отброшены. Затем отбрасывается ведущая семёрка или восьмёрка, если пользователь набрал номер с кодом страны, и строка обрезается до десяти цифр. После очистки номер собирается заново: к префиксу «+7» по мере накопления цифр добавляются скобки и дефисы. Поскольку строка формируется поэтапно с проверкой длины, маска корректно отображается и при частично введённом номере.

### 2.6.3 Автоматическое сокращение слов в адресе

Поле адреса дополнено сценарием, который приводит ввод к принятому в адресных базах сокращённому виду. Сценарий хранит словарь соответствий: «улица» заменяется на «ул. », «микрорайон» — на «мкр-н. », «квартал» — на «кв-л. ». При каждом изменении поля регулярное выражение `/^([А-Яа-яЁёA-Za-z]+)\s/` выделяет первое слово, завершённое пробелом; слово приводится к нижнему регистру, чтобы замена срабатывала независимо от того, с какой буквы его набрал пользователь, и, если оно найдено в словаре, заменяется сокращением.

Дополнительно сценарий следит за регистром названия улицы: после того как пользователь поставил запятую (то есть закончил вводить название), первая буква названия автоматически делается заглавной. Для этого используется регулярное выражение с Unicode-классом `\p{L}` и флагом `u`, корректно работающее с кириллицей, — классический класс `\w` русские буквы не распознаёт.

### 2.6.4 Проверка формата адреса при отправке формы

Последний сценарий выполняет контроль формата адреса непосредственно перед отправкой формы на сервер.

```javascript
document.getElementById('form-adress').addEventListener('submit', function (e) {
    const pattern = /^ул\.\s*.+,\s*\d+[А-Яа-яA-Za-z]*$/;
    if (!pattern.test(addressInput.value.trim())) {
        e.preventDefault();
        alert('Адрес должен быть в формате: ул. Название, 12 или 12А');
    }
});
```

Рисунок 2.10 — Проверка формата адреса перед отправкой формы

Обработчик события `submit` проверяет значение поля регулярным выражением, которое требует строгой структуры: сокращение «ул.», название улицы, запятая и номер дома, состоящий из цифр с необязательной буквой (например, «12А»). Если адрес не соответствует шаблону, метод `e.preventDefault()` отменяет отправку формы, а пользователю выводится подсказка с примером правильного формата. Такая проверка на стороне клиента экономит обращение к серверу: заведомо некорректный адрес, по которому база данных всё равно ничего не нашла бы, отсеивается ещё в браузере. При этом адрес в базе хранится именно в сокращённом формате, поэтому связка «автозамена + проверка формата» существенно повышает вероятность успешного поиска задолженности.

## Выводы по главе

В ходе работы над клиентской частью сайта была выполнена вёрстка трёх страниц с использованием семантических тегов HTML5. Раскладка элементов построена преимущественно на флексбокс-модели, для ряда одинаковых по ширине кнопок применена грид-сетка. Оформление вынесено в единый файл стилей: определены классы шапки, кнопок, форм, модальных окон и карточек задолженностей, выдержанные в общей чёрно-белой цветовой гамме. Адаптивность под мобильные устройства обеспечена сочетанием функции `clamp()` и медиазапроса, перестраивающего страницу в одну колонку на экранах уже 700 пикселей. Клиентские сценарии на JavaScript реализуют управление модальными окнами, маску телефонного номера, автоматическое сокращение адреса и его проверку перед отправкой, что снижает количество ошибочных запросов к серверу и упрощает ввод данных для пользователя.
